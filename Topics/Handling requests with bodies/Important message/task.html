This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

In this task, you have to implement a simple service that stores one message.

The program should contain the following two endpoints:

  • POST /message — receives a JSON object with one field importantMessage (String) and stores the message.
  • GET /message — returns the newest message as a JSON object with one field importantMessage (String).

In both cases, the server should respond with 200 OK status code.

Example 1: POST /message request with the following body:

{
   "importantMessage": "a28957asdf792347f",
}

Example 2: GET /message request

Response:

{
   "importantMessage": "a28957asdf792347f",
}

Example 3: POST /message request with the following body:

{
   "importantMessage": "zxcvlj243243l",
}

Example 4: GET /message request

Response:

{
   "importantMessage": "zxcvlj243243l",
}
