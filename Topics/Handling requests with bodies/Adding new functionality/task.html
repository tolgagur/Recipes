This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

In this task, you need to improve the program that stores data. Currently, the program has one endpoint:

  • GET /api/data/{id} — returns data with a specified id as a JSON object with one field data (String).

Your task is to analyze the code and add a new functionality:

  • POST /api/data/new — receives a JSON object with one field data (String), then generates a unique integer id and puts the received data together with generated id into the map. After that, the program should return a JSON object with one field id. The status code should be 200 OK.

Example 1: POST /api/data/new request with the following body:

{
   "data": "something1",
}

Response:

{
   "id": 1
}

Example 2: POST /api/data/new request with the following body:

{
   "data": "something2",
}

Response:

{
   "id": 2
}

Example 3: GET /api/data/1 request

Response:

{
   "data": "something1",
}

Example 4: GET /api/data/2 request

Response:

{
   "data": "something2",
}
